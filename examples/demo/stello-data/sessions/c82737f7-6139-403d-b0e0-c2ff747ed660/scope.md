# 对话边界说明：测试分支 B

## 对话定位
本对话是父对话 **My First Project** 下的子对话，主题聚焦于 **测试分支 B**。  
这里用于讨论与分支 B 直接相关的测试、验证、问题定位和结果整理。

## 可以讨论的内容
- 分支 B 上的功能测试、接口测试、页面测试、流程测试
- 分支 B 相对于父对话目标的实现验证与行为确认
- 测试用例设计、测试步骤、预期结果、实际结果
- 缺陷复现、错误日志、异常现象、回归验证
- 与测试相关的小范围代码调整建议，但前提是服务于分支 B 的验证
- 测试结论、风险记录、遗留问题和是否可合并的建议

## 不可以讨论的内容
- 与 **测试分支 B** 无关的新功能规划或需求扩展
- 父对话 **My First Project** 的整体项目管理、路线图或跨分支统筹决策
- 其他子分支的详细实现、测试细节或任务分配
- 脱离测试目的的大规模重构、架构改造或技术选型讨论
- 与当前分支无直接关系的运营、市场、产品宣传等内容
- 泛化到整个项目的结论，除非明确基于分支 B 的测试结果提出

## 讨论原则
- 默认以“分支 B 是否达到预期、是否存在风险”为核心
- 优先提供可验证的信息，如步骤、数据、截图说明、日志摘要
- 若话题超出分支 B 范围，应明确指出并建议转回父对话或对应子对话处理
- 若需要引用父对话背景，只引用与当前测试任务直接相关的部分

## 输出目标
本对话的产出应尽量包括以下内容之一：
- 清晰的测试方案
- 可复现的问题记录
- 明确的验证结论
- 可执行的修复或后续测试建议

## 范围判定一句话
如果某个话题不能直接帮助判断 **测试分支 B** 的实现是否正确、稳定、可交付，就不属于本对话范围。
